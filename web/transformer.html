Transformer 的本质是个 "下一个 token 预测器"：给一段文本，输出词表上每个词作为下一个词的概率，采一个出来，再接着预测。它能爆火的核心是 self-attention——一次前向就能把任意远的词关联起来。下面用 GPT-2 (small) 为例，把这套机制拆开看。

01整体流水线 可交互

从一句话到下一个词，数据要走这 6 段。前两段是 tokenizer 的活（无参数、纯规则），后面才是模型（有参数、会训练）。看一个 token 走完全程——到终点吐出"下一个词"，再拼回句尾重新出发（这就是自回归，详见 §07）。

分界线：模型只认整数 id，不认字符串 Tokenizer 是一套 BPE 规则表 + 词表，把文本切成子词再查成 id 列表；模型从拿到 id 开始，W_E（词向量）和 W_P（位置向量）都是可学习参数。同一套 tokenizer 能给多个模型用。

03Q / K / V：借书单类比

为什么要拆成三个角色？因为"我在找什么"和"我能被找到什么"不是一回事——同一个人，写借书单和印书脊标题，内容不一样。

12 个头的"分工"是训练逼出来的，不是写死的

这 12 套矩阵初始全是随机数、毫无区别。但它们的输出会拼起来一起用——如果都学同一件事就浪费了 11 个头。训练时梯度会自然把它们推向不同方向。事后探查发现：有的头专看"前一个词"，有的专看句末标点，有的专看代词指代关系。

类比：雇 12 个新人不分配岗位、只考核团队总产出，半年后他们自己分出了采购/销售/客服——因为这样 loss 最低。头的数量（12）是人设的超参数，头之间做什么是学出来的。

05一个 Transformer Block 在干什么

输入输出形状都是 (n, 768)——token 数没变、维度没变，变的是向量里的"内容"。一个 Block 做两件事：

① Attention — 开会

在 token 维度操作 · 跨 token 混合

所有 token 坐一起交换信息，每个词按相关度吸收其他词的上下文。这是唯一让 token 之间"互动"的环节。

② MLP — 会后整理

在特征维度操作 · 每个 token 独立

每个 token 单独把刚听到的消化成新理解：768→3072→GELU→768，先扩张到高维做非线性混合再压回。模型的事实知识主要存在这里。

对比维度在哪个维度操作功能token 之间有互动吗
Attentiontoken 维度（跨 token）混合上下文信息有
MLP特征维度（各算各的）非线性特征加工无
外面还包着两层"保护壳" LayerNorm：把每层输出拉回差不多的数值尺度，防止越传越飘（GPT-2 用 Pre-LN，放子层之前）。残差连接 x' = x + Sublayer(LN(x))：让梯度沿加号直通流回底层，相当于给反向传播开高速公路。两者缺一训练就跑不起来。

12 个这样的 Block 串起来。浅层学词法语法，深层学语义推理：苹果 → 商业语境的苹果 → 一家科技公司 → 库克掌管的那家。

06从向量到下一个词

MLP 之后，每个 token 都是一个加工好的 768 维向量。但我们要的是一个词，不是向量。这中间发生了三件事：① 只取最后一个 token → ② LM Head 给词表里每个词打分 → ③ softmax 转概率、采样。一步步看。

① 为什么只用"最后一个" token 的向量？ 可交互

常见疑问：注意力不是每个词都和所有词算吗，为啥说"只看左边"？ 关键在 因果掩码（causal mask）。算注意力会先得到一张 n×n 分数矩阵，本来每个词确实能看到所有词；但 GPT 在 softmax 之前，把"看右边（未来）"的格子全置成 −∞，softmax 后变成 0 —— 这条通路被掐断了。

为什么要掐？因为任务是"预测下一个词"。训练时整句一起喂进去，如果位置 love 能看到右边的答案 you，它直接抄答案、啥也学不会。所以强制：预测第 i 个词时只能看 0~i，训练和推理才一致（推理时未来的词本来还没生成）。双向、不掩码的是 BERT 那类 encoder。

于是每个位置"看到的范围"是阶梯式的，只有最后一个位置看到了整句话——它的 768 维向量才是"读完全文后对下一个词的预判"，所以只送它去打分：

（训练时不一样：每个位置都要预测它的下一个词，所以 n 个位置全都打分、一次算 n 份 loss。但推理生成时只关心最后一个。）

② LM Head：一个向量，怎么变成 5 万个分数？

先破一个误解：h 不是"最后一个词"的含义 读完 苹果的发布会让股价___，若 h 只代表"股价"，最像它的词该是"股票/价格"——可我们要的是"大涨"。经过 12 层加工、又只有最后位置读完了整句，h 代表的是"读完全文后对'下一个词该长什么样'的预判"——一个理想答案的向量。打分就是拿它去词表里找最贴合的真实词。

具体怎么打分：LM Head 是一个 768 × 50257 的大矩阵——它的每一列就是一个词的"特征向量"。拿 h 和每个词的列向量做点积：方向越接近，分数（logit）越高。

"点积大" = "连贯的下一个词"，是训练焊进去的 语义连贯是目标，点积只是媒介。几十亿次完形填空同时拧两边的螺丝：让 h 指向连贯续写的方向，也让"该接上的词"的向量正好落在那个方向上。拧到最后，"点积大"就被定义成了"训练数据里最该接上的词"。

把词义压缩到只有 3 维（真实是 768 维，原理一样），每维代表一种语义属性：①科技/商业 ②食物/水果 ③动作/动词。于是每个词就是 3 个数：

h 这根箭头指向"科技/商业"，于是和同方向的词点积大、得分高；和"食物""动作"方向的词点积小。 这就是 logit。

为什么这套机制能学会？三点 ① 点积天生是相似度尺子（|h|·|词|·cos 夹角，和 §02 的 Q·Kᵀ 同一把尺）； ② 词向量和"产生 h 的权重"一起被梯度训练，逼到该出现的词正好落在 h 的方向上； ③ 近义词向量相近，h 指对方向，一整簇近义词同时得高分。
一句话记住：调收音机 h 是模型喊出的"暗号频率"，5 万个词各守一个频率，点积就是"谁共振最强"——共振最强的词就是最可能的下一个词。
最后一个 token 的向量 · h  ·  和词表里每个词的向量做点积 → logit
点积大 = 方向接近 = "这个词和当前语境很搭" → logit 高。这里只画了几个候选；真实是对全部 50,257 个词同时打分，得到 50257 个 logit。
省参数小技巧：Weight Tying（权重绑定） GPT-2 直接把输入端的词嵌入矩阵 W_E 转置过来当 LM Head 用——"查向量"和"按向量打分"复用同一张表，省下 3800 万参数，效果还更好。

③ Softmax → 概率 → 采样

50257 个 logit 还只是"分数"（有正有负、不归一）。softmax 把它们压成和为 1 的概率：分数高的概率大。最后按概率采一个词出来——注意：不是简单粗暴取分数最高的那个（那只是 temperature→0 的特例），而是按概率随机采，所以同一句话每次续写可能不同。怎么采，决定输出风格。拖动温度看分布怎么变：

三种采样策略 Temperature：T<1 分布更尖、输出保守确定；T>1 更平、发散有创造力；T→0 退化成贪心。 Top-k：只从概率最高的 k 个里采。 Top-p（核采样）：取累计概率刚超过 p 的最小集合里采，比 top-k 更自适应。
07自回归循环 → 为什么有 Prefill / Decode 可交互

拿到下一个词，拼回末尾，再走一遍

朴素做法每生成一个 token 就重算整个序列所有层的 Q/K/V——O(n²)，越写越慢。解法是 KV Cache：把历史 token 的 K、V 缓存下来，下一步只给新 token 算一份，和缓存拼起来做注意力，decode 每步降到 O(n)。

这正是实际推理分两阶段的原因：Prefill 一次性算完输入所有 token、把 KV 灌进缓存（算力重）；Decode 逐 token 生成、复用缓存（访存重）。

08GPT-2 (small) 的数字

Transformer = 嵌入 → N 层（attention 混 token + MLP 变特征 + 残差 + LN）→ 投影到词表 → 采样。
所有"智能"——Q/K/V 的视角、多头的分工——都不是人写死的规则，而是给定结构后，让模型在海量数据上做完形填空，梯度下降自己涌现出来的分工。
